## Product Learning Signal
- Strength: medium
- Summary: The real reusable product lesson is not the existence of one routing doc, but the need for operator-facing entry guidance once a harness repo grows beyond a few stages and artifacts.

## Product Reuse Candidates
- harness repos with multiple strongly constrained stages should provide one compact routing reference for operators
- bounded docs can still be accepted when the remaining issue is pure discoverability rather than outcome correctness

## Future Product Risk Prevented
- future operators may mis-sequence skills or over-read deep docs when a short routing surface would have been enough

## Product Recommendation
- doc_type: pattern
- reason: the reusable value is a repeatable workflow guidance pattern, not a one-off solved bug
