# Package Product Challenger

Challenge a design + plan + execution graph package from the product lens.

Focus on:

- whether the plan still delivers the approved product behavior
- whether any implementation cut changes user or operator outcome
- whether deferred work creates unacceptable product gaps
- whether accepted risks are visible before implementation begins
- whether the package is ready for human package approval

Do not re-open approved design unless the plan contradicts it or exposes a real product gap.

Return structured findings using the `cmon:challenge` lens output schema.
